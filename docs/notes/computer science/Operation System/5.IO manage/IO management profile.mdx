## IO设备的分类
:::note
**以信息交换的方式分类**
- 块设备: 信息传输以数据块为单位，属于结构设备，例如磁盘（磁盘 **传输速率高**，**可随机读写**）
- 字符设备: 信息交换以字符为单位，属于无结构类型，例如交换式终端机，打印机等（基本特征是 **传输速率低**，**不可寻址**，**一般采用中断IO方式传输**）。
:::
:::note
**按传输速率分类**
- **低速设备**: 传输速率为每秒几字节到数百字节的设备，例如键盘鼠标。 
- **中速设备**: 传输速率为每秒数千字节到数万字节的设备，例如激光打印机。
- **高速设备**: 传输速率在数百千字节到千兆字节的设备，比如磁盘机、光盘机。
:::
## 设备控制器（IO接口）
设备管理器位于CPU与设备之间，既与CPU通信又和设备进行通信，还要按CPU发来的命令去控制设备的功能
![](https://skywalkerch-1303839378.cos.ap-nanjing.myqcloud.com/mypicbed202307110836064.svg)
### 设备控制器的组成
1. **CPU与控制器接口**：包括数据线、地址线、控制线；数据常与两类寄存器相连（1、数据寄存器：存放从设备送来的输入数据或来自CPU的输出数据。2、控制/状态寄存器：存放从CPU送来的控制信息或者设备的状态信息）。
2. **控制器与设备接口**：一个设备控制器要连接多个设备，因此其内部有多个**设备与控制器接口**，每个接口都有**数据**、**状态**、**控制**这三种信号。
3. **IO逻辑**：用于实现对设备的控制，**IO逻辑**通过一组控制线与CPU交互，对从CPU收到的IO命令进行译码。CPU启动设备时，CPU会将启动命令发送给控制器，同时通过地址线把地址送给控制器，控制器的**IO逻辑**译码后，选择相应的设备进行控制。
### 设备控制器的功能
:::note
- 接收并识别CPU发来的命令
- 数据交换，包括设备和控制器之间，控制器与内存之间的数据交换
- 标识或报告设备的状态，用于CPU处理
- 地址识别
- 数据缓冲
- 差错控制
:::
### IO端口
IO端口是设备控制器中能被CPU直接访问的控制器
- 数据寄存器：实现CPU与设备之间的数据缓冲
- 状态寄存器：获取执行结果和设备的状态信息，让CPU知道是否有准备好
- 控制寄存器： 由CPU进行写入，以便启动命令或更改设备模式
为了访问这些寄存器，一般有**独立编址** （`对IO端口编号，用特殊的指令访问`）和**统一编址**（`将数据存于内存固定区域，用内存访问指令访问`）进行访问
